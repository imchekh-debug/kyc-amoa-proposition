  • Réglementaire : Respect des obligations LCB-FT (identification clients/bénéficiaires effectifs, vigilance constante/allégée), gel des avoirs et reporting TRACFIN, avec conservation des justificatifs 5 ans minimum. Risques de sanctions ACPR ou remédiations coûteuses en cas de non-conformité.
  • Opérationnel : SI fragmenté générant backlogs, ressaisies et délais excessifs ; besoin de transversalité pour métiers (banque, financement, investissement) et clients variés (collectivités, entreprises, investisseurs).
  • Stratégique : Automatisation pour réduire fraudes documentaires, améliorer UX client (taux d'abandon <10% via digitalisation) et productivité (gain 30-50% temps traitement via OCR/API).

Attentes concrètes :

  • Cadrage : Définition périmètre (fonctionnel : workflows KYC, scoring risque ; technique : API avec CRM/AML ; organisationnel : gouvernance multi-équipes), estimation coûts (TCO 5 ans : 1-3 M€ CAPEX + OPEX licences), planning (9-24 mois total), risques (juridique : non-auditabilité ; réputationnel : scandales ; opérationnel : retards intégration).
  • Marché public : Rédaction CCTP (exigences fonctionnelles/techniques/réglementaires), grille scoring (pondération 40% fonctionnel, 30% technique, 20% coûts, 10% risques), dépouillement offres.
  • Déploiement : Pilotage agile (SCRUM/KANBAN), suivi tickets/backlog, ateliers validation, formation (manuels utilisateurs), reporting (KPI : avancement, risques, adoption).
  • Livrables : Note cadrage, CCTP, benchmark, plan déploiement, tableaux bord, rapport final validation.
